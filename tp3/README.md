# TP3 : Maquette graphique

> Remise : 12 juin\
> Modalité : en équipe de 2

Tout au long de la session, vous travaillerez en équipe de deux à la
refonte d'un site Web de votre choix. Chaque travail pratique (TP)
correspondra à l'un des éléments livrables du processus de conception
d'une interface Web : charte graphique, maquette conceptuelle, maquette
graphique et intégration.

Pour ce troisième TP, votre tâche consiste à concevoir les maquettes
graphiques pour la refonte des pages Web sur lesquelles vous avez
travaillé pour le TP2.

## Consignes

- Pour chacune des pages, choisissez une des deux maquettes
  conceptuelles que vous avez faites pour le TP2. Alternativement, vous
  pouvez combiner les deux maquettes.
- Pour chacune des maquettes conceptuelles choisies, concevez une
  maquette graphique pour la version _desktop_ du site Web, et une autre
  maquette pour la version _mobile_ (pour un total de dix maquettes
  graphiques).
- Vos maquettes graphiques doivent respecter les règles de la charte
  graphique conçue pour le TP1. Ces règles peuvent être mises à jour au
  besoin.
- Les différents états de l'interface doivent être représentés : menus
  ouverts et fermés, boutons et liens en survol, etc.
- Le contenu textuel et visuel doit représenter la version finale du
  site Web. Vous ne pouvez pas utiliser de paramètres fictifs
  (_placeholders_).

> [!NOTE]
> Vos maquettes graphiques devront être intégrées tel quel pour le TP4,
> alors ne soyez pas trop ambitieux·ses. C'est OK de ne pas tout
> inclure.

## Remise

Votre travail doit être remis sous la forme d'un document Figma qui
inclut les composants et les maquettes. Ajoutez un lien vers le document
Figma dans le README du dépôt Classroom créé pour le TP1.

## Présentation

Lors de la remise, vous devrez présenter votre travail à la classe (et
aux parties prenantes imaginaires). Votre présentation ne doit pas
dépasser 10 minutes, et votre caméra doit être ouverte durant celle-ci.
Pendant la présentation :

- expliquez votre choix de maquettes conceptuelles ;
- présentez les maquettes graphiques en comparant votre refonte à la
  version originale du site Web ; et
- expliquez les changements apportés à la charte graphique (s'il y a).

## Critères d'évaluation

- Maquettes [7 points]
  - Le document Figma est bien organisé.
  - Les règles de la charte graphique sont respectées.
  - Une seule et même grille est utilisée pour toutes les maquettes.
  - Les éléments de l'interface sont bien alignés sur la grille.
  - Les éléments de l'interface récurrents sont abstraits en composants.
  - Les composants sont définis indépendamment des maquettes.
  - Les composants sont nommés et leur nom exprime bien ce qu'ils
    représentent.
  - Les éléments de l'interface sont mis en forme avec la disposition
    automatique.
  - Les propriétés de composant sont utilisées lorsque c'est pertinent.

- Présentation [3 points]
  - Les changements sont bien expliqués.
  - La présentation est soignée et bien structurée.

Outre ces critères, vous pouvez être pénalisé·es si votre travail ne
respecte pas les consignes, ou si la charge de travail n'est pas divisée
également.
